Ext Server - Simple Web Server

only call "Ext.application", and a Web server starts. describe the next code, and any file can start a server.

We often use the name called "server.js" for a file name to describe Ext.application.

can call the return value of require('ext-server') directly to become the Ext object.

require('ext-server').application();

The document root of the Web server is by default "public" directory. and the index that this file is stored away by "public" is html.

When you access http://localhost:8124 by a browser, index can see html.

Server Settings

The setting of the server specifies Object Literal in an argument of Ext.application and performs it.

The default value is the same as the next specification.

Ext.application({
    name    : 'App',
    port    : 8124,
    cluster : false,
    onLaunch: Ext.emptyFn
    app     : {
        // application setting objects.
    }
});

"name" is the high-end name of the namespace that your application uses.

A worker of the number of the CPU cores starts when I set "cluster" in true and works as a kernel base load balancer.

Please specify the number of the workers with a number not true when you specify the number of workers.

onLaunch is the callback function that is called after listen of the server was completed. Not the request from a client, only once is called at the time of the start of the server.

"app" does the working setting of the application framework.
